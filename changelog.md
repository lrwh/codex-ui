# Changelog

按日期记录 `Codex for Linux` 的主要功能变更和结构调整。

## 2026-04-27

- 新增应用内版本与更新能力：
  - 引入统一版本文件 `VERSION`
  - 启动后后台自检查最新 Release
  - 左侧版本号在存在新版本时显示“有更新”
  - 设置面板支持“检查更新”和“下载更新”
- 更新检查从 GitHub API 切换为 `releases/latest` 重定向方案，规避匿名限流导致的更新失败。
- 下载更新时优先拉取 `.deb` 安装包，并将安装命令复制到剪贴板，同时打开下载目录。
- 打包链路改为从 `VERSION` 读取版本号，保证应用内版本、`.deb` 包版本和发布 tag 一致。
- 发布 `0.3.0` 版本，并上传 `.deb` 与 `tar.gz` 安装产物。

## 2026-04-26

- 新增左侧会话状态标识：
  - 正在运行的会话显示 `运行中`
  - 后台收到新回复但尚未点开的会话显示 `未读`
- 修复多会话并行时“当前会话”状态串台问题，右上角 `running / idle` 只反映当前选中会话。
- 修复切换会话后工作目录未同步更新的问题，当前会话卡片会跟随 session 切换刷新 `cwd`。
- 调整左侧品牌区，仅保留产品名和版本号，移除“Linux 桌面客户端”副标题。
- 发布 `0.2.0` tag 和 GitHub Release，用于验证更新能力前的基线版本。

## 2026-04-25

- 优化 README 结构：
  - 将构建说明拆分到独立的 `build.md`
  - README 新增安装方式和启动方式说明
- 完善发布文档，补充 `.deb` 安装、目录包启动和桌面入口注册方式。
- 提交并推送一轮会话状态处理与文档整理改动。

## 2026-04-24

- 21:10 提交代码：更新 desktop_app_window_layout.py

- 12:03 提交代码：新增 package_deb.sh；更新 desktop_app_ui.py, desktop_app_window.py, desktop_app_window_common.py, desktop_app_window_conversation.py 等 6 项

- 支持输入框直接从剪贴板粘贴图片，并自动落盘为图片附件。
- 支持在输入框里直接粘贴文件管理器复制的本地文件，作为 `.log`、`.md`、图片等附件加入当前输入。
- 修复已有会话 `resume + 图片附件` 时的命令参数顺序问题，避免出现 `No prompt provided`。
- 将产品展示名称调整为 `Codex for Linux`，同步到窗口标题和界面主品牌展示。
- 增加会话级工作目录能力：
  - 新会话默认使用全局 `work_dir`
  - 已有会话默认沿用 session 文件里的历史 `cwd`
  - 支持通过“改路径”按钮为当前会话单独覆盖工作目录
- 将工作目录展示从左侧品牌区移动到“当前会话”标题后面，并对长路径做截断显示。

## 2026-04-23

- 初始化桌面客户端项目，建立基于 PySide6 的主界面、会话列表、当前会话、消息区、输入区和状态区。
- 支持读取本地 `~/.codex` 会话数据，浏览历史会话并继续对话。
- 支持多账号管理，包括账号切换、账号用量刷新、剩余额度与刷新时间展示。
- 支持会话操作，包括置顶、重命名、复制会话 ID、打开会话文件、复制 `codex resume` 命令和本地别名持久化。
- 支持长会话优化，默认仅渲染最近消息，并按批加载更早内容。
- 支持附件输入，文本附件内容自动拼入 prompt，图片附件走 Codex 图片输入链路。
- 增加快捷指令模板、权限模式切换、`/model` 模型选择和推理强度设置。
- 增加 `Ctrl+Enter` 发送、`Ctrl+N` 新会话、`Esc` 停止当前请求等快捷操作。
- 修复多会话并行输入、账号跳变、线程退出崩溃、中文输入、消息区域空白、错误信息丢失等稳定性问题。
- 完成 Linux 桌面打包链路，包括 PyInstaller 配置、启动脚本、桌面入口和应用图标。
- 优化 README，增加预览截图和功能说明。
- 将单文件桌面客户端拆分为多个 Python 模块，移除旧的 Go 终端版代码。
